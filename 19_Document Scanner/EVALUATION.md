## 2. 케이스 분류

### 성공 케이스 (기본 파라미터, 15장)

기본 파라미터(blur=5, Canny 50/150)로 즉시 성공한 이미지들의 공통 특성:
- 문서와 배경 사이 밝기 차이가 뚜렷함 (흰 종이 vs 회색/어두운 배경)
- 문서의 4개 변이 모두 프레임 안에 완전히 포함됨
- 엣지가 끊김 없이 연속적으로 이어짐

대표 예시: images\_(4) — 소파 위 정면 촬영. 천 배경과 흰 종이의 대비가 명확하여 기본 Canny로 4변 모두 깨끗하게 검출.

### 부분 성공 케이스 (폴백 전략 필요, 5장)

기본 파라미터로는 실패했으나 다른 전략으로 복구된 이미지들.

**images\_(8) — 파라미터 조정으로 복구**
- 상황: 소파 위, 소파 줄무늬 패턴이 문서 경계와 유사한 엣지를 생성
- 기본 실패 원인: Canny high=150이 문서 엣지와 소파 패턴을 동시에 검출하여 사각형 근사 실패
- 해결: blur=7로 올려 소파 패턴을 흐리게 하고, Canny를 30/100으로 낮춰 문서 경계만 남김

**images\_(12) — Morphology 보강으로 복구**
- 상황: 문서가 세워져 있고 약간 구부러져 있음
- 기본 실패 원인: 구부러진 상단 변이 Canny에서 끊어져 닫힌 윤곽선을 형성하지 못함
- 해결: dilate(2회)로 끊긴 엣지를 연결한 뒤 erode(1회)로 두께를 복원하여 닫힌 사각형 형성

**images\_(18) — Morphology 보강으로 복구**
- 상황: 카펫 바닥 위 작은 문서
- 기본 실패 원인: 카펫 텍스처가 수많은 미세 엣지를 생성하여 문서 윤곽선이 묻힘
- 해결: Morphology 보강으로 미세 엣지를 병합하고 문서의 큰 윤곽선만 남김

**images\_(2) — 적응형 임계값으로 복구**
- 상황: 프린터 위 문서, 프린터 본체가 어두운 회색
- 기본 실패 원인: 문서(어두운 인쇄물)와 프린터 본체의 밝기 차이가 너무 작아 Canny가 경계를 놓침
- 해결: 적응형 임계값(Adaptive Threshold)으로 지역적 대비를 강화한 뒤 Canny를 적용하여 흰 여백과 어두운 인쇄 영역의 경계에서 문서 윤곽을 추출

**images\_(3) — 확장 epsilon으로 복구 (가장 어려운 케이스)**
- 상황: 프린터 위 문서, 기울어짐, 하단 변이 프린터 급지구에 묻힘
- 기본 실패 원인: 문서 하단 변이 프린터 본체와 겹쳐 4개 꼭짓점 검출 불가. 표준 epsilon(0.02)으로는 근사 실패
- 해결: epsilon을 0.03으로 확장하고, 극저 임계값(Canny 20/60) + 강한 dilate(2회)를 적용하여 미약한 엣지까지 연결. 종횡비 검증(1.1~2.0)으로 오검출 방지

## 3. 파라미터 딜레마

### 모든 이미지에 적용 가능한 최적 파라미터가 존재하는가?

**존재하지 않는다.** 아래 표가 이를 보여준다.

| 파라미터 | images\_(4) 결과 | images\_(8) 결과 | images\_(3) 결과 |
|---------|-----------------|-----------------|-----------------|
| blur=5, Canny 50/150 | 성공 | 실패 | 실패 |
| blur=7, Canny 30/100 | 성공 | 성공 | 실패 |
| blur=3, Canny 20/60 | 실패 (노이즈 과다) | 실패 | 성공 |

images\_(4)는 대비가 좋아 대부분의 파라미터에서 성공하지만, images\_(8)은 블러를 올려야 소파 패턴을 제거할 수 있고, images\_(3)은 반대로 블러를 낮추고 임계값을 극저로 내려야 미약한 엣지를 잡을 수 있다.

한 이미지의 최적 파라미터가 다른 이미지에서 실패하는 이유: 각 이미지의 배경 텍스처, 조명 조건, 문서-배경 대비가 다르기 때문에 "노이즈를 제거하면서 동시에 약한 엣지를 보존하는" 단일 균형점이 존재하지 않는다.

## 4. 규칙 기반의 한계

### 설계한 규칙 목록

| # | 규칙 | 역할 |
|---|------|------|
| 1 | 가우시안 블러로 노이즈 제거 | 미세 엣지 억제 |
| 2 | Canny 이중 임계값으로 엣지 검출 | 문서 경계 추출 |
| 3 | 면적 필터링 (전체 이미지 5% 이상) | 작은 노이즈 윤곽선 제거 |
| 4 | approxPolyDP로 꼭짓점 4개 사각형 검출 | 문서 형태 판별 |
| 5 | x+y 합 / y-x 차로 좌표 정렬 | 투시 변환 입력 좌표 순서 보장 |
| 6 | Morphology(dilate/erode)로 끊긴 엣지 보강 | 폴백 전략 |
| 7 | 적응형 임계값으로 저대비 이미지 전처리 | 폴백 전략 |
| 8 | 종횡비 검증 (1.1~2.0) | 오검출 방지 |

### 각 규칙이 실패하는 상황

| 규칙 | 실패 상황 |
|------|-----------|
| 블러 + Canny | 배경 텍스처와 문서 대비가 유사하면 구분 불가 |
| 면적 필터링 | 문서가 작으면 (이미지 5% 미만) 필터에 걸려 제거됨 |
| 꼭짓점 4개 검출 | 구겨진 문서, 모서리가 접힌 문서는 4개 이상의 꼭짓점 생성 |
| 좌표 정렬 (합/차) | 극단적으로 기울어진 문서에서 좌상/우상 판별이 모호해질 수 있음 |
| Morphology 보강 | 팽창이 과하면 인접 물체와 문서 엣지가 병합됨 |
| 종횡비 검증 | 정사각형에 가까운 문서(명함 등)는 필터에 걸릴 수 있음 |

### "규칙을 추가하면 해결되는가?"

부분적으로만 해결된다. 이번 테스트에서 폴백 전략(규칙 추가)으로 75% → 100%를 달성했지만, 다음과 같은 근본적 한계가 있다.

첫째, 전략이 추가될수록 처리 시간이 증가한다. 확장 epsilon 전략은 파라미터 조합을 수백 가지 시도하므로 기본 대비 수십 배 느려진다.

둘째, 규칙 간 충돌이 발생한다. 텍스처 배경에서 노이즈를 제거하려고 블러를 올리면 약한 문서 엣지도 사라지고, 약한 엣지를 살리려고 임계값을 낮추면 노이즈가 폭증한다. 이 딜레마는 규칙 추가로 해결할 수 없다.

셋째, 예상하지 못한 새 조건에 무방비다. 이번 20장에서는 성공했지만, 구겨진 문서, 반투명 문서, 빛 반사가 심한 문서 등 새로운 조건이 나타나면 또 다른 규칙을 수동으로 추가해야 한다.

## 5. 학습 기반 접근이 필요한 이유

규칙 기반 접근에서 경험한 핵심 문제는 "사람이 예상한 조건에서만 동작한다"는 것이다.

20장의 이미지에서 5가지 서로 다른 전략을 설계했는데, 각 전략은 특정 실패 케이스를 분석하고 사람이 원인을 파악한 뒤 수동으로 만든 것이다. 만약 1,000장, 10,000장의 다양한 이미지에 대응해야 한다면, 사람이 모든 실패 조건을 예상하고 규칙을 설계하는 것은 불가능에 가깝다.

학습 기반 접근은 이 문제를 다음과 같이 해결할 수 있다.

"어떤 파라미터가 최적인가?"를 사람이 결정하는 대신, 대량의 이미지-정답 쌍에서 모델이 자동으로 최적의 특징 추출 방법을 학습한다. 예를 들어 딥러닝 기반 문서 검출(U-Net, YOLO 등)은 수만 장의 문서 이미지를 학습하여 배경, 조명, 각도에 관계없이 문서 영역을 세그멘테이션할 수 있다.

이번 미션에서 직접 경험한 "파라미터 딜레마", "규칙 간 충돌", "새 조건에 대한 무방비"는 모두 규칙 기반 접근의 구조적 한계이며, 이것이 데이터에서 학습하는 방식이 필요한 근본적인 이유이다.
